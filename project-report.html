1. 项目概述

目标

使用 LoRA (Low-Rank Adaptation) 技术对通义千问 Qwen2.5-1.5B-Instruct 模型进行指令微调，使模型在通用中文问答场景下表现更好。项目全程使用国产云端 GPU 完成训练，本地仅负责数据准备和代码管理。

技术栈

Python 3.13 PyTorch 2.10 LLaMA-Factory LoRA (rank=16) Qwen2.5-1.5B CUDA 12.8 AutoDL Cloud HuggingFace Datasets Gradio

4. 训练流程

第 1 步
项目规划 & 代码搭建
分析需求（知识注入任务），设计项目结构，编写 8 个核心脚本文件，确保可复用、可配置。
第 2 步
数据集准备
从 HuggingFace 下载 shibing624/alpaca-zh 数据集，通过 prepare_dataset.py 转换格式并切分为 80% 训练集 + 20% 验证集。
第 3 步
AutoDL 云端环境配置
注册 AutoDL，创建 NVIDIA RTX 5090 (32GB) 实例。将项目文件上传至 /root/autodl-tmp/，执行 cloud_setup.sh 一键安装 LLaMA-Factory 及所有依赖。
第 4 步
50 条小规模验证训练
使用 50 条样本验证 pipeline 可运行，12 步训练耗时 6.6 秒，loss 从 1.524 降至 1.416，确认配置正确。
第 5 步
全量正式训练
39,054 条数据 × 3 epochs = 7,323 步训练，总耗时约 73 分钟。生成 3 个 checkpoint（7000、7200、7323 步）。
第 6 步
模型评估 & 聊天测试
在云端使用 llamafactory-cli chat 加载 LoRA 权重进行交互式对话测试，验证微调效果。
第 7 步
下载产物 & 上传 GitHub
下载 LoRA 适配器权重到本地 output/，将项目完整代码推送到 GitHub 仓库。

7. 踩坑 & 解决方案

由于国内网络环境限制和云端镜像版本差异，实际过程中遇到了 6 个问题，均已解决：

⚠ 问题 1：HuggingFace 数据集下载失败
✅ 解决：本地用户环境 SSL 不可用 → 改用 uv pip install datasets 管理依赖 → 数据在本地准备后上传云端。
⚠ 问题 2：GitHub 克隆被墙
✅ 解决：AutoDL 实例无法访问 github.com → 通过 ghproxy.com 镜像克隆，或改为直接 pip install llamafactory。
⚠ 问题 3：torchaudio 库与 CUDA 版本不兼容
✅ 解决：pip uninstall torchaudio -y → 文本微调不需要 torchaudio → 修改源码让 import 变为可选。
⚠ 问题 4：datasets 版本过高被拒绝
✅ 解决：设置环境变量 DISABLE_VERSION_CHECK=1 跳过版本检查。
⚠ 问题 5：配置文件参数不兼容新版 LLaMA-Factory
✅ 解决：移除 dataset_info、logging_dir 字段，将 warmup_ratio 改为 warmup_steps。
⚠ 问题 6：模型从 HuggingFace 下载被墙
✅ 解决：改为从 ModelScope (modelscope.cn) 下载模型 → train_config.yaml 中指定本地路径。

9. 总结

本项目从零到一完成了一次完整的 LoRA 微调流程，核心成果包括：

  • 可复用的项目框架：scripts/ 下的 7 个脚本覆盖了数据准备 → 训练 → 评估 → 部署的全链路，适用于任何 HuggingFace 数据集

  • 云端最佳实践：ModelScope 替代 HF、hf-mirror.com 加速、DISABLE_VERSION_CHECK 跳过版本冲突等实用方案
